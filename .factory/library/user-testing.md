# User Testing

Testing surface, tools, setup steps, and known quirks.

**What belongs here:** how to launch the app/service, what real surfaces to test, manual validation notes, and current limitations.

---

- Manual on-device validation is mandatory because capture depends on live macOS permissions and cross-app behavior.
- Test from a real logged-in macOS session, not only unit tests.
- Minimum user-testing surfaces for this mission:
  - first-launch onboarding and permission setup
  - a supported non-secure text input context
  - a degraded/limited app context
  - a secure/password field
  - provider settings in Local and remote/custom modes
  - Daily Story generation after same-day writing
- Verify that product wording remains honest about best-effort capture.
