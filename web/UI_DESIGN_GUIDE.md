# TypeLearn UI Design Guide

## Style Definition

The reference points to a **soft editorial learning UI**:

- editorial, not dashboard-heavy
- playful, but not childish
- tactile, but not skeuomorphic
- airy, but still warm

For TypeLearn, the right adaptation is:

> **Soft Editorial Atelier**

It mixes four ingredients:

1. **Storybook edtech**
   Clean composition, generous whitespace, oversized headings, and learning-oriented copy.
2. **Pastel marker accents**
   Short bursts of mint, butter, peach, rose, and sage used like highlighted notes.
3. **Tactile paper surfaces**
   Cream backgrounds, cocoa text, framed panels, and buttons that feel pressed instead of glowing.
4. **Selective glass**
   Keep glass only where it adds calm layering: sidebar, top bar, toolbars, and key hero panels.

## What This Is Not

- not blue/purple SaaS gradients
- not neon dark-mode glassmorphism
- not dense metrics-first productivity UI
- not “AI dashboard” aesthetics

## Visual Principles

### 1. Warm paper first

The default canvas should feel like cream paper in daylight, not a white app shell.

- base background: warm oatmeal / parchment
- surfaces: ivory, vellum, light sand
- text: espresso / cocoa, never pure black

### 2. Color is used like annotation

Accent colors should behave like highlights, tabs, stickers, and inline emphasis.

- sage
- mint
- butter
- peach
- dusty rose
- clay

Use them in small moments. Do not flood the whole UI with one loud accent.

### 3. Glass survives, but in a restrained form

Glass is still part of the product identity, but it should become **frosted vellum**, not glossy candy.

- subtle blur
- warm translucent fill
- pale border
- soft inner highlight
- low-contrast shadow

Good places for glass:

- sidebar shell
- sticky top bar
- filter/date toolbar
- page hero panels

Bad places for glass:

- every card
- every list item
- every nested section

### 4. Typography should feel designed

Use a strong serif display paired with a clean modern sans and a compact mono for labels.

- display: expressive serif
- body: rounded, readable sans
- utility labels: restrained mono

Headlines should feel editorial and memorable, not generic product copy.

### 5. UI density should breathe

TypeLearn is ambient software. The UI should never feel crammed.

- larger vertical rhythm
- fewer hard dividers
- cleaner grouping
- obvious hierarchy between hero, controls, and content

## Component Rules

### Navigation

- floating frosted rail
- active item uses warm contrast, not saturated color
- badges are tiny and low-noise

### Hero sections

- big, editorial headline
- one sentence that explains the value
- pastel inline highlight pills
- small metrics framed as notes, not analytics tiles

### Artifact cards

- paper cards instead of glass cards
- light tonal differentiation between Expression and Refinement
- clear internal sections: original, result, why, patterns

### Buttons

- tactile, outlined, slightly raised
- no electric glows
- hover should feel sharper, not brighter

### Empty states

- calm and useful
- explain what will appear here
- no generic “nothing here” dead ends

## Motion

- use subtle lift and opacity transitions
- no bouncing
- no glowing pulse
- no floating gradients

## Palette Direction

### Core neutrals

- parchment
- oat
- ivory
- cocoa
- warm taupe

### Accent family

- sage
- mint
- butter
- peach
- dusty rose
- clay

### Forbidden legacy direction

- bright blue
- indigo
- violet
- purple gradients

## Product Translation

TypeLearn should now feel less like a generic AI workspace and more like:

- a quiet language atelier
- a learning notebook with good taste
- a private studio for turning raw typing into better expression

That is the target for this redesign.
